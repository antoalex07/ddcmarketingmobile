Create `src/screens/AppointmentCreateScreen.js` — a minimal form for creating appointments offline. This is Phase 1 scope: just enough to create an appointment that saves locally. Phase 2 will add full validation, client search/selection, and inline client creation.

**Imports:**
```javascript
import React, { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, StyleSheet, Alert, ScrollView, Platform } from 'react-native';
import { SafeAreaView } from 'react-native-safe-area-context';
import { useAuth } from '../context/AuthContext';
import { insertAppointment } from '../db/appointmentDB';
```

**Form fields (minimal viable set):**
- `clientName` (TextInput) — free text for now, Phase 2 adds client type selection and search
- `clientType` (simple picker or 3 TouchableOpacity buttons) — 'doctor', 'clinic', or 'hospital'
- `appointmentDate` (TextInput with placeholder YYYY-MM-DD) — simple text input for Phase 1, Phase 2 will add DatePicker
- `timeFrom` (TextInput with placeholder HH:MM) — simple text for Phase 1
- `timeTo` (TextInput with placeholder HH:MM) — simple text for Phase 1
- `notes` (TextInput, multiline) — optional

**Client type selector:** Three buttons in a row, highlight selected. Default to 'doctor'.

**Submit handler (`handleCreate`):**
1. Basic validation: clientName, appointmentDate, timeFrom, timeTo must not be empty. If any empty, show Alert with friendly message ("Please fill in all required fields").
2. Build appointment data object:
```javascript
const appointmentData = {
  appoint_userid: user?.userid || user?.id,
  appoint_staffname: user?.staffname || user?.name || '',
  appoint_assigneduserid: user?.userid || user?.id,
  appoint_assignedstaffname: user?.staffname || user?.name || '',
  appoint_clientname: clientName.trim(),
  appoint_clienttype: clientType,
  appoint_clientautoid: 0,    // No client ID in Phase 1, Phase 2 adds client selection
  appoint_clientaddress: '',   // Phase 2 adds address from client record
  appoint_appointmentdate: appointmentDate.trim(),
  appoint_timefrom: timeFrom.trim(),
  appoint_timeto: timeTo.trim(),
  appoint_notes: notes.trim(),
  appoint_status: 1,           // 1 = Scheduled
  appoint_crtdby: user?.staffname || user?.name || '',
  appoint_crton: new Date().toISOString(),
};
```
3. Call `insertAppointment(appointmentData)` from appointmentDB
4. On success: show Alert "Appointment created! It will sync when you're online." then `navigation.goBack()`
5. On error: show Alert "Couldn't save appointment. Please try again." (friendly, no technical details per user decision)

**Styling:** Match existing app style conventions (look at AppointmentUpdateScreen.js for colors, spacing, font sizes). Use the blue (#2563eb) primary color for the submit button. Input fields with border, rounded corners, padding.

**IMPORTANT constraints:**
- Do NOT add a DatePicker or TimePicker component — Phase 2 handles that. Use plain TextInput for dates/times.
- Do NOT add client search/autocomplete — Phase 2 handles that.
- Do NOT add location/GPS capture — workers just type what they need for now.
- Keep it simple: this screen exists so workers CAN create appointments offline. Phase 2 makes it polished.
